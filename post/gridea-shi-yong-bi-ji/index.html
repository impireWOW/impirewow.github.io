Gridea使用/备份笔记

· 2022-02-12 · # Gridea # 环境配置 # 工具

Gridea工作目录默认位置是在Documents/Gridea下
想自定义也是可以的,它会在选定目录下初始化一组网站文件配置
如果遇到要更换工作平台的情况,将Gridea工作目录下的/config文件夹和/posts文件夹备份即可 ,
自己想git push 就直接把Gridea工作目录下
如果要导入之前写好的外部.md文件到./posts,不可以直接复制 ,可以按照Gridea的md文件格式在文件开头加入所需参数 ,或者在软件里直接新建一个同名的md文件,然后去编辑器里直接复制过来就可以了

git的配置都在/config下,没有加密,是好还是坏呢(

theme的选择不如hugo hexo那么多 但是也够用,选择的时候记得测试一下本地的访问速度
本地速度不行的话就干脆换掉得了,像github这种国内访问比较差的平台本来就卡,要加载的资源多了更会GG

git push之后如果页面没变不要着急,给它一点时间反应,这个和平台关系不大,就是正常的延迟
备份也可以借助网盘服务来自动备份(
