哈希表,也叫散列表,是集合结构之一

我们都知道有这些常用的数据结构:数组 链表 二叉树 哈希表
大致列出一下这些数据结构的插入,查找 ,删除等操作的性能对比

数组 线性链表 二叉树 哈希表
查找 O(1)/O(n)/O(logn) O(1) O(logn) O(1)
插入/删除 O(n) O(n) O(logn) O(1) 
注:
对于指定下标的查找，时间复杂度为O(1)；通过给定值进行查找，需要遍历数组，
逐一比对给定关键字和数组元素，时间复杂度为O(n)，
对于有序数组，则可采用二分查找，插值查找，斐波那契查找等方式，可将查找复杂度提高为O(logn)

可见,相较于其他数据结构,哈希表在增删改查操作的时候速度最快,因此许多缓存技术的核心也是在内存中维护一张大的哈希表

哈希表的主干是数组,用当前元素的关键字借助哈希函数映射到数组的某个位置,通过数组下标一次性定位即可完成操作,采用链地址法,即数组+链表的方式进行存储
